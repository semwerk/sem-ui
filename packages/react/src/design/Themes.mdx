# Theme System

Werk design system provides multiple theme variants for different products and contexts.

## Available Themes

### Base Themes
Default themes for general use across all Werk products.

- **base.light** - Clean, accessible light theme
- **base.dark** - Low-contrast dark theme

### Product-Specific Themes

#### Werkcontext Themes
Optimized for development tools and code documentation.

- **werkcontext.light** - Enhanced readability for code contexts
- **werkcontext.dark** - Developer-friendly dark theme

#### WerkPress Themes
Designed for content-focused documentation sites.

- **werkpress.light** - Reader-optimized light theme
- **werkpress.dark** - Comfortable dark reading experience

## Theme Structure

Each theme provides:
- **Semantic colors** - Success, warning, error, info states
- **Background layers** - Default, subtle, muted
- **Text hierarchy** - Default, muted, subtle, inverse
- **Border variants** - Default, subtle, strong
- **Component tokens** - Button, input, card styling

## Switching Themes

Use the Storybook toolbar to toggle between light and dark backgrounds and see how components adapt.

## Implementation

Themes are CSS files that set CSS custom properties:

```css
/* base.light.css */
:root {
  --color-bg-default: #ffffff;
  --color-text-default: #212529;
  --color-border-default: #dee2e6;
  /* ... */
}
```

## Usage

Import the desired theme CSS in your application:

```tsx
import '@werk-design/css'; // Includes all themes
// or
import '@werk-design/css/base.light.css'; // Specific theme
```
